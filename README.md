## Background

IBM DOORS (Database for Object-Oriented Requirements Specification) has been an industry-standard requirements management tool since the 1990s. The modern incarnation — **DOORS Next** (part of the **Engineering Lifecycle Management** suite) — supports OSLC-based integration with test management, configuration management, and CI/CD pipelines.

For a systems architect, the critical value of DOORS is its ability to create a **digital thread**: an unbroken chain of traceability from stakeholder need → system requirement → design artefact → test case → test result.

## Version Coverage

This repository covers both:
- **IBM DOORS Classic** (v9.x) — still widely deployed in defence and aerospace
- **IBM DOORS Next / ELM** (v7.x) — the cloud-capable successor
