# Daily Learning Guide — IBM DOORS ELM

A 10-day structured learning programme for systems architects introducing automated testing with IBM DOORS ELM.

## Pedagogy Approach

This guide applies **Bloom's Taxonomy** — the most widely validated framework for instructional design — to structure learning from foundational recall through to applied creation.

### The 10-Day Arc

| Days | Bloom's Level | Focus |
|------|--------------|-------|
| 1–3 | Remember → Understand | Core concepts: traceability, requirements quality, DOORS architecture |
| 4–6 | Understand → Apply | Operationalising: attributes, links, ETM integration |
| 7–8 | Apply → Analyse | Scripting, baselines, configuration management |
| 9–10 | Evaluate → Create | CI/CD integration, PDF import, building your own workflow |

### Each Day Follows the Same Structure

Each lesson is designed for approximately **90 minutes**:

| Section | Time | Purpose |
|---------|------|---------|
| Learning Objectives | 2 min | Prime your brain — set expectations |
| Core Concept | 15 min | Reading — key ideas with examples |
| Key Definitions | 5 min | Spaced repetition glossary |
| Practical Exercise | 30 min | Hands-on task to reinforce the concept |
| Connection to Automated Testing | 5 min | Explicit link to your automation goal |
| Reflection Prompts | 5 min | Consolidation — write 2–3 sentences |
| Further Reading | — | 2–3 curated links for deeper exploration |
| Tomorrow's Preview | 2 min | Priming — reduces cognitive load for the next session |

### Why These Methods?

- **Bloom's Taxonomy** ensures you don't just read about DOORS — you build towards applying and creating with it.
- **Spaced repetition** (key definitions repeated daily) fights the forgetting curve.
- **Explicit automation connection** each day keeps the learning relevant to your goal.
- **Reflection prompts** activate metacognition — you learn better when you explain what you've learned.
- **Tomorrow's preview** uses the *generation effect* — priming a concept the day before you study it significantly improves retention.

## Prerequisites

- Basic understanding of requirements engineering concepts (what a requirement is)
- Access to an IBM DOORS Next or DOORS Classic environment (a trial/sandbox is sufficient)
- Familiarity with systems engineering terminology (stakeholder, artefact, traceability)

No prior DOORS experience is required.

---

## Tips for Success

1. **Do one day at a time** — spaced practice beats cramming. Return the next day.
2. **Complete the exercise** — reading without doing produces shallow learning.
3. **Write your reflections** — even two sentences. Externalising knowledge cements it.
4. **Skim the next day's preview** the evening before — it takes 2 minutes and significantly helps.
